[译]如何在Archlinux上安装Yaourt

九月 12, 2017 发布在 archlinux

9月一次日常系统升级后，导致Remarkable无法使用。重装了也没解决，最后发现是依赖的webkitgtk版本过旧，被官方标记为不安全，从仓库中移除。这过程中，Yaourt和AUR也被我翻来倒去地折腾，差点被玩坏。所以还是备份一下Yaourt的安装方式。

Arch Linux与其他主要发行版的一个主要不同点是允许用户能够根据喜好安装来自社区仓库(Arch User Repository or Simply AUR)的软件包。这是一个完全由社区驱动的软件仓库，其中包含了约44,000多个软件包。 然而，用户无法使用自带的pacman包管理器来安装这些第三方软件。只能自己下载，校验后再安装。Yaourt(Yet AnOther User Repository Tool), 用于便利地下载AUR仓库中的软件。 简而言之，Yaourt是在Pacman上提供了AUR软件的安装和其他一些新的功能。

在这篇博客里，我会介绍在Arch上安装Yaourt的两种方式:

  1. 通过他人的仓库安装Yaourt
  2. 利用AUR安装Yaourt [注]第一种方式会更容易一些

1. 通过他人的仓库安装Yaourt

打开并编辑Pacman的配置文件:

sudo vim /etc/pacman.conf

在文件末尾增加一个新的仓库源:

[archlinuxfr]
SigLevel = Never
Server = http://repo.archlinux.fr/$arch

保存修改。

最后， 更新仓库内容并安装Yaourt

sudo pacman -Sy yaourt

这样就安装好了，接下来就可以直接使用Yaourt进行包的管理了。用法和Pacman相同，可以通过-h参数查看命令的帮助信息。

2. 利用AUR安装Yaourt

在正式安装Yaourt之前，需要先安装一些依赖的包。执行：

sudo pacman -S --needed base-devel git wget yajl

在完成这些基本依赖后，我们需要下载并安装package-query来构建和运行Yaourt。 可以通过Git获得package-query的代码：

git clone https://aur.archlinux.org/package-query.git

clone完成后，进入package-query文件夹，编译并安装：

cd package-query/
makepkg -si

稍等片刻，Package-query就安装成功了。接下来，我们需要clone最新的Yaourt代码：

git clone https://aur.archlinux.org/yaourt.git

同样，编译并安装Yaourt：

cd yaourt/
makepkg -si

至此，Yaourt就安装完成了。

[注]还可以安装aurvote小工具，给你喜欢的软件包投票打分。
